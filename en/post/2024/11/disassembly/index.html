Posted on 2024-11-03 Disqus:

DCU Secure Programming Course Disassembly Guide

Due to the repetitive patterns of the exercises, using a fixed approach can lead to fast problem-solving.

Prerequisites

  1. Familiarity with various assembly instructions Assembly Instructions
  2. Understanding of % and $ for representing registers and immediate values $ and % - Registers and Immediate Values
  3. Familiarity with direct and indirect addressing Direct and Indirect Addressing
  4. Understand a case study Example of C Code to Assembly

Problem-Solving Approach

  1. Find the number of parameters
  2. Identify the number of local variables
  3. Identify the loop body
  4. Parse remaining code segments
  5. Identify return values

Step 1: Find the Number of Parameters

ebp is the saved frame pointer, and ebp + 4 is the return address. Since the parameters are usually int or int *, ebp + 8, ebp + 12, ebp + 16 positions represent the first, second, and third arguments, respectively.

To determine the number of parameters quickly, look for patterns like 0x__(%ebp) in the code and find the largest offset. (offset - 4) // 4 will give you the number of parameters.

Loop bodies are usually implemented as while or for loops.

For common while loops, look for:

  • Loops:
    Loops contain an unconditional jmp instruction that points to the start of the condition check.

  • Condition:
