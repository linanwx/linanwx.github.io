个人博客从 Hugo 替换成 Hexo

这周开始，我把博客从 Hexo 迁移到 Hugo，主要是因为：

Hugo 对 i18n 支持更好。因为 Hexo 对 i18n 并不太支持，之前自己魔改了模板，最后实现了中文和英文切换，但是维护成本很高，了解后发现 Hugo 原生支持多语言。

不过在切换后，发现也有以下一些优点：

主题配置更简单。我使用的是Stack 主题，基本上没有遇到坑。官方自带的 example 配置就很好，照这配置就好。另外，还带 Google Analysis 分析，还带 RSS 源，源也会自动区分语言，非常舒适。

在切换的过程中，基本上没有遇到什么障碍，只要有你过去的 Markdown 文件，直接复制到新工程的目录下，再强制推送到原来的 github pages仓库，就可以直接迁移过来。强制推送后，需要在 github重新配置一下自定义域名，就无损升级了，包括评论之类的都是一样的。

唯一不同的是，Hugo 默认使用小写链接，而 Hexo 默认保留大小写，这可能使得过去的一些链接失效，搜索引擎需要重新建立索引。

神秘模型Optimus Alpha出现在https://openrouter.ai/ 并霸榜

由于该模型免费，并且性能很强，在上线后，就被选择成为 OpenRouter 的首席流量处理模型。

随着14 号 OpenAI 官方公布 GPT4.1 后，该模型从 OpenRouter 下架。显然，该模型就是 GPT4.1。

GPT4.1 是一个编程能力特化的模型，拥有 1M 的上下文窗口。确实，现在编程领域，OpenAI 被 Claude 3.7 还有 Gemini2.5pro 打的鼻青脸肿，无论是 o1 还是 o3 模型，都没能在编程领域取得什么领先地位。希望 OpenAI 再接再厉吧。

OpenRouter 是一个让用户统一接入多个大语言模型 API 的中间层服务平台。你可以把它理解成一个「代理」或「路由器」，统一管理不同大模型厂商（OpenAI、Anthropic、Mistral、Google Gemini、Cohere 等）的调用方式。

看起来，这次，OpenAI 选择了 OpenRouter 作为测试发布平台，而不是https://lmarena.ai/

OpenRouter.ai

OpenRouter

这是一个 LLM平台，包含大量的免费/收费高质量模型。

选择 Prompt pricing 为 0 元，再选择 Top Weekly，就可以看到目前最好的免费模型。

不过使用时需要开启 log 和使用数据进行训练。

目前可以选择 Gemini 2.5 pro。

其中Ranking功能可以实时查看平台上所有的流量分发情况，哪个模型好用一目了然。

nanobrowser

一款用于 Chrome 的浏览器插件，可以控制浏览器的自动化操作。

试验了一下，无法支持 Arc 浏览器。体验下来能力有限，例如，对网页的滚动偶尔失效，无法总结整个页面的内容，只能看到局部的内容，操作偶尔不正确。

虽然该产品不太推荐，不过这可能是未来一个方向，看起来，以后会出现大量的 AI 代理。

抢票、刷分之类的灰色产业估计会很开心。

markitdown

一款用于将不同格式的内容转换成 Markdown 格式的工具。

离线情况下，可以处理不同的文件转换成 Markdown 格式，试用了一下，转换得比较基础。

也可以支持接入 LLM，提升转换结果和文档中图片的部分。

在未来可能是一款不错的 MCP 服务。

PDF search

一款应用，可以快速搜索多个 pdf 中出现的内容。

使用类似模糊搜索的方式，原理是对 PDF 做了分割并且进行了嵌入向量计算，随后和搜索词进行匹配。

适合考试期间搜索文档使用。

缺点是，index 是按照页面划分的，搜索后需要自己在页面中找到对应的 part。
